1. 북마크 서비스의 인덱스 페이지 UI를 구현해 줘

# 페이지 구성

- 상단 헤더
  - 좌측 : 초코프레스 텍스트 로고
  - 우측 : + 새 링크 버튼
- 좌측 사이드 바
  - All 버튼 배치2
  - 그 아래에는 폴더 리스트 배치
- 메인 섹션
  - 그리드 등록된 링크들을 배치

2. 새 링크 페이지 UI를 구현해줘

- 페이지 주소 /new 설정
- 인덱스 페이지의 + 새 링크 버튼 클릭하게 되면 이 페이지로 이동하게 해 줘

# 페이지 구성

- 상단 헤더, 좌측 사이드바 인덱스 페이지와 동일
- 메인 섹션
  - 링크를 입력하는 인풋
  - 폴더를 선택하는 셀렉트 박스
  - 저장 버튼

3. 폴더 별 링크 페이지의 UI를 구현해 줘

- 페이지 주소 /folder/폴더아이디
- 사이드바에서 폴더를 클릭하면 이 페이지로 이동하게 해줘

4. 폴더 추가 기능을 만들어 줘

- 새 폴더 버튼을 헤더 컴포넌트의 새 링크 버튼 좌측에 배치해줘
- 새 폴더 버튼 클릭 시, 화면 가운데 새로운 폴더를 생성하는 모달 창 열림
  - 폴더 이름 인풋
  - 취소, 저장 버튼
- 새롭게 추가된 폴더는 좌측 사이드바에 배치되어야 한다.

5. 폴더 삭제 기능 구현해줘

- 삭제 버튼을 좌측 사이드바의 폴더 이름에 마우스를 올렸을 때 폴더 이름 우측에 나타나게 해줘
- 삭제 버튼은 휴지통 모양의 아이콘을 설정해
- 삭제 버튼 클릭 시, 모달 창을 열어서 진짜 삭제할 건지 다시 한번 확인해

6. 폴더 수정 기능을 추가해줘

- 폴더 이름에 마우스를 올렸을 때 폴더 이름 우측에 나타나게 해줘
- 폴더 수정 버튼은 옅필 모양의 아이콘로 설정해
- 수정 버튼 클릭시에는 폴더 이름을 수정하는 모달 창을 열어 줘

7. 링크 관리 기능 구현해 줘

- 먼저 특정 링크의 오픈그래프 정보를 추출하는 api를 구현해
- 새 링크 페이지에서 확인 버튼을 누르면 링크를 저장해, 오픈 그래프 api를 통해서 링크의 핵심 정보인 제목, 설명, 썸네일, 링크 주소를 함께 수집해서 저장해줘
- 새롭게 보관되는 링크 카드에 썸네일도 함께 표시해 줘

8. 링크 삭제 기능을 구현해 줘

- 삭제 버튼 링크 카드에 마우스를 올리게 되면 우측 상단에 휴지통 모양의 버튼으로 나타나게 해 줘
- 삭제 버튼 클릭시 확인을 구하는 모달 창을 열어 줘

9. 링크 수정 기능을 만들어줘

- 수정 버튼을 링크 카드에 마우스를 올렸을 때 우측 상단에 연필 모양의 아이콘으로 나타나게 해 줘
- 수정 버튼 클릭시 링크 정보를 수정할 수 있는 모달 창을 열어 줘
- 모달 창 안에는 링크의 폴더, 제목, 설명만 수정할 수 있도록 해 줘

# 주의사항

- 컴포넌트 단위로 적절히 분할해서 구현
- 결과는 컴포넌트 단위로 숫자 리스트로 보고해
- 이미 만들어둔 컴포넌트를 최대한 활용해

* supabase table 생성

1. 링크 테이블을 생성해줘

- 컬럼 : id, url, title, description, thumbnail_url, created_at
- id 컬럼에 오토 인크리먼트, 프라이머리 키 설정
- id, url, created_at 컬럼은 필수 컬럼으로 설정, 나머지는 선택적 컬럼으로 설정해
- RLS는 꺼 줘

2. 폴더 테이블을 생성해 줘

- 컬럼 : id, name, created_at
- id컬럼에 오토 인크리먼트, 프라이머리키를 설정 해
- 모든 컬럼을 필수 컬럼으로 설정
- RLS를 꺼 줘

3. 링크 테이블과 폴어 테이블을 연결해 줘

- 링크 테이블에 folder_id라는 컬럼을 추가해 줘
- folder_id 컬럼은 선택적 컬럼으로 설정해 줘
- folder_id 컬럼이 folder 테이블의 id 컬럼을 참조하도록 외래키로 설정해 줘
- folder테이블의 데이터가 삭제될 경우(폴더 삭제) 연결된 link 테이블의 folder_id를 빈 값으로 비워 줘

4. 폴더 추가 기능을 구현 해

- 사용자가 추가한 폴더를 folders테이블에 추가해
- 폴더 추가 버튼의 중복 클릭을 막아서, 여러개의 폴더가 동시에 추가되지 않도록 해 줘
- 좌측 사이드바에 folders 테이블에 보관된 폴더들을 불러와서 순서대로 나열해 줘
- 이 외의 시키지 않은 기능까지는 굳이 구현하지마
